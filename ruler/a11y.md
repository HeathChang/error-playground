# Accessibility (a11y)

> `base.md`를 상속한다. WCAG 2.1 AA 수준 준수를 목표로 한다.
> 이 프로젝트는 프레임워크 독립 라이브러리다 — 예시는 **바닐라 DOM/HTML** 기준으로 쓴다(React/JSX 아님).

## 이 프로젝트의 a11y 불변식 (최우선)

> **어떤 경험(게임·애니메이션)을 띄우든, 상태코드 + 핵심 메시지 + 홈/뒤로 링크는 항상 접근 가능해야 한다.**
> 이 불변식이 깨지면(예: 게임 캔버스가 폴백 링크를 덮어 포커스 불가) 설계 실패다. (`docs/PLAN.md` §5.4·§14)

## 기본 원칙

- 모든 상호작용 요소는 **키보드로 조작 가능**해야 한다.
- 시각적 정보는 **대체 텍스트**로 동등하게 제공한다.
- 색상 대비: 일반 텍스트 **4.5:1 이상**, 큰 텍스트(18pt 이상 또는 14pt bold) 및 UI 컴포넌트·그래픽 경계는 **3:1 이상** (WCAG 2.1 AA).

## prefers-reduced-motion (이 프로젝트 필수)

- 애니메이션/게임은 `prefers-reduced-motion: reduce` 를 **반드시** 존중한다.
  - 자동 재생 애니메이션은 정지/축소하고, 게임은 **자동 시작하지 않고 사용자 opt-in(버튼/키 입력)** 으로만 시작.
  - 근거: 전정기관 장애 사용자에게 과한 모션은 메스꺼움·현기증을 유발. 에러 페이지는 사용자가 *원치 않게* 도달한 곳이라 더더욱 강요 금지.
- 코어가 시스템 값을 읽어 `ExperienceContext.reducedMotion` 으로 확정 전달하므로, 경험은 이 값만 보고 분기한다.

```ts
// DO — 모션 선호도 존중
const reduce = window.matchMedia('(prefers-reduced-motion: reduce)').matches;
if (reduce) {
  showStartButton();      // 자동 재생 대신 명시적 시작
} else {
  startAnimationLoop();
}
```

```css
/* DO — CSS 차원에서도 가드 */
@media (prefers-reduced-motion: reduce) {
  .ep-anim { animation: none; transition: none; }
}
```

## 시맨틱 마크업

- 역할에 맞는 HTML 태그를 사용한다 (`header`, `main`, `nav`, `button`, `a`).
  - 근거: 시맨틱 태그는 스크린 리더가 *지역으로 점프*하는 단축키(랜드마크)를 제공. div로만 짠 화면은 점프 불가 = 전체 순차 탐색 강제.
- `div` + 클릭 핸들러로 버튼을 만들지 않는다. `<button type="button">` 또는 `<a href>` 를 쓴다.
  - 근거: div는 Tab 포커스 불가, Enter/Space 키 처리 없음, 스크린 리더가 "button"으로 안 읽음 — 키보드/스크린 리더 사용자 완전 배제.
- 제목 레벨(`h1~h6`)은 **건너뛰지 않는다**. 에러 페이지 제목은 `h1` 하나로 시작.

## 필수 속성

| 요소 | 필수 |
|------|------|
| `<img>` / SVG | `alt` 또는 `role="img"`+`aria-label` (장식용은 `alt=""` / `aria-hidden="true"`) |
| 아이콘 전용 버튼 | `aria-label` |
| 게임 `<canvas>` | `role="img"` + `aria-label`(정적 설명) 또는 인접한 텍스트 대체. 점수 등 동적 정보는 `aria-live` 영역으로 별도 노출 |
| 동적 알림(점수·게임오버) | `aria-live="polite"` |

## 캔버스/게임 접근성

- `<canvas>` 자체는 스크린 리더에 불투명하다. **게임은 부가물(enhancement)**, 핵심 정보(에러·링크)는 캔버스 밖 실제 DOM에 둔다.
- 게임 조작은 키보드로도 가능해야 한다(스페이스/방향키 등). 터치 전용 금지.
- 점수/상태는 `aria-live` 텍스트로도 갱신해 비시각 사용자에게 전달.

## 키보드 내비게이션

- Tab 순서는 시각적 순서와 일치.
- 포커스 링을 숨기지 않는다 (`outline: none` 후 대체 스타일 필수).
- 게임이 키 입력(스페이스 등)을 가로채더라도 **Tab으로 폴백 링크에 도달 가능**해야 한다(포커스 트랩 금지).

## AI 행동 규칙

- 클릭 가능한 `div`/`span` + 이벤트 리스너 발견 시 즉시 `<button type="button">`/`<a>` 로 교체.
- `<img>`/장식 SVG 생성 시 `alt`/`aria-hidden` 누락 금지.
- 아이콘만 든 버튼 발견 시 `aria-label` 추가 강제.
- 애니메이션/`requestAnimationFrame` 루프 작성 시 `prefers-reduced-motion` 가드 없으면 코드 작성 금지.
- ARIA는 *마지막 수단* — 시맨틱 태그가 같은 역할을 표현 가능하면 ARIA 사용 금지.
  - 근거: 잘못 쓴 ARIA는 무 ARIA보다 나쁘다. 시맨틱 태그가 옳은 역할을 *암묵적으로* 부여한다.

## 패턴 (DO / DON'T)

### 클릭 요소 (바닐라)

```ts
// DON'T — 키보드 접근 불가, 시맨틱 부재
const el = document.createElement('div');
el.addEventListener('click', goHome);

// DO
const btn = document.createElement('a');
btn.href = '/';
btn.textContent = '홈으로';
```

### 포커스 스타일

```css
/* DON'T — 키보드 사용자가 현재 위치를 잃음 */
button { outline: none; }

/* DO — 기본 outline 제거 시 대체 스타일 제공 */
button { outline: none; }
button:focus-visible { outline: 2px solid var(--ep-color-focus); }
```

### 기타 금지/권장

| DON'T | DO |
|-------|-----|
| 게임 캔버스가 폴백 링크를 덮어 포커스 불가 | 폴백은 캔버스 밖 DOM, 항상 도달 가능 |
| 모션 강제 자동 재생 | `prefers-reduced-motion` 시 opt-in 시작 |
| 색상만으로 상태 전달 | 색상 + 아이콘 + 텍스트 |
| 클릭 가능한 `div` | `<button>` / `<a>` |
